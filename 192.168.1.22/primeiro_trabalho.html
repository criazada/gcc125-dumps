2.1 - Serviço de sincronização de tempo(NTP)

Após a configuração e login no software OpenVPN conforme explicado no campus virtual, foi instalado o software PuTTY para acessar as máquinas fornecidas pelo professor, com os seguintes IPs: 192.168.1.22 e 192.168.1.23. Ao efetuar o login, a primeira ação realizada foi trocar a senha das duas VMs, de “aluno” para “12345”.

Para a sincronização de tempo foi instalado o pacote ntpdate utilizando a linha de comando “sudo apt install ntpdate” nas duas VMs. A primeira VM, de IP com término em 22, foi escolhida para prestar serviço de horas para a segunda, de IP terminado em 23. O serviço de horas escolhido para a primeira VM foi o “pool.ntp.org” e utilizando o comando “sudo ntpdate -u pool.ntp.org”, tornou-se possivel a sincronização com o horário do serviço de horas pool. Posteriormente seguindo os mesmos passos foi possível sincronizar as horas do segundo VM com o primeiro, utilizando o comando “sudo ntpdate -u 192.168.1.22”.

2.2 - Servidor WEB

Para servir de servidor web foi escolhida a primeira VM de com IP de 192.168.1.22. Já para a criação do servidor web foi escolhido utilizar o Apache por disponibilizar mais documentações. Sendo assim, com comando “sudo apt install apache2” foi instalado o apache na VM.

Apos a instalação do Apache foi criado um arquivo de configuração para o servidor, na pasta sites-available localizada no diretório de instalação do Apache. Utilizando o comando “sudo cp 000-default.conf soalegria.conf”gerou-se um arquivo de configuração com o nome “www.soalegria.ufla.br” para o servidor.

Posteriormente, foi criado uma pasta para o domínio no diretório “/var/www/” com o nome de soalegria, e tambem, um arquivo index.html. O site criado servirá de repositório para todos os trabalhos realizados na disciplina, para sua construção foi utilizado o software VScode pela facilidade de edição e visualização. Para finalização, foi realizado a migração para o servidor web.

2.3 - Teste do servidor WEB

Para testar o servidor web foi instalado na segunda VM um software de browser para linha de comando chamado w3m, utilizando o comando “sudo apt -y install w3m”. Após a instalação do mesmo, foi utilizado o comando “w3m 192.168.1.22” para acessar o servidor da primeira VM. O Acesso foi realizado com sucesso e foi visualizado o arquivo html criado no tópico 2.1.
